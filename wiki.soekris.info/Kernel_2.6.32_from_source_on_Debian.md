# Kernel 2.6.32 from source on Debian

## Debian Squeeze kernel package for net6501

This is a lightweight kernel config for Debian Squeeze that still includes support for anything one would want to run on the net6501. It should work for i386 kernels and be easily adapted to amd64 builds.

Due to the differences between the net6501 and the other Soekris boards, using the older kernel configs is not an ideal starting point. This configuration started with the standard debian 2.6.32 config, and then everything that definitely is not needed on a net6501 was disabled.

Trimming down has not been brought to the extreme, there are still many unneeded modules enabled, however it is a good starting point that leads to a kernel package that is about one third the size of the regular package.


The trimmed config also shortens compile times significantly, especially when compiling on the Soekris itself. Comments and suggestions for improvement appreciated.

[Download config-2.6.32-net6501](https://web.archive.org/web/20120221052819/http://u.delta9.pl/k/soekris/config-2.6.32-net6501.txt) or [Debian Squeeze 2.6.32-net6501 i386 kernel packages](https://web.archive.org/web/20120221052819/http://u.delta9.pl/k/soekris/)

### Wanted Hardware Support

The hardware spec sheets are still not complete on the net6501 so there might still be some hardware on the Soekris that isn't supported on this kernel. However everything that was supported in the standard Debian kernel should be included in this config.

In particular, I have yet to get enough details about 

```
 * hardware watchdog
 * programming the GPIO pins / FPGA
 * Real Time Clock
```

We will need kernel 2.6.37 for proper support:

### How to Recompile the Package

If you wish to reconfigure the packages with added or removed support, or maybe to get an amd64 package, I include the method here for completeness:

#### build dependencies

```
# apt-get install kernel-package linux-source-2.6.32
# cd /usr/src
# tar xf linux-source-2.6.32.tar.gz
# cp config-net6501 linux-source-2.6.32/.config

# cd linux-source-2.6.32

if you want to change anything
  # make menuconfig 

else to configure the sources
  # make oldconfig  
```

#### building the kernel package

```
# export CONCURRENCY_LEVEL=2
# make-kpkg clean
# suffix=brigaden
# revision=8
# fakeroot make-kpkg --append-to-version "-$suffix" --revision "$revision" \
  --us --uc --initrd kernel_image kernel_headers
```

#### installing the package

```
# cd .. && dpkg -i linux-kernel-2.6.32-$suffix_$revision_$arch.deb

```

Kernel kompiles can be done on a different host, provided the same architecture and distribution. Just copy the config file and follow compile-kernel.sh
